# Planning Documents

This directory contains all planning, architecture, and implementation-design documents.

## Structure

- `architecture/rag-system-design.md` - north-star system architecture for the organisational RAG platform.
- `phase/phase1-mvp-plan.md` - Phase 1 MVP execution plan and module-level implementation baseline.
- `implementation/supabase-pgvector-playground-plan.md` - staged plan for adding Supabase pgvector support to the playground track.
- `implementation/playground-llm-rag-plan.md` - plan for adding LLM-backed answer synthesis to the playground query flow.

## Usage

- Start with `architecture/rag-system-design.md` for design intent.
- Use `phase/phase1-mvp-plan.md` for build-order and delivery scope.
- Use files under `implementation/` for track-specific execution details.
